## 収益化の設計

### 現在の状態（A型・1枚HTML）
```
index.html
└── data.json → カードに「Official Site ↗」ボタン
                ↑ ここを将来アフィリエイトリンクに差し替える
```

### Google AdSense 配置計画（申請後）
```
場所①：検索フィルターの下（横長バナー 728x90）
場所②：カードグリッドの間（4件ごとに1枚インフィード広告）
場所③：フッター上（横長バナー）
```

### Amazon アフィリエイト差し替え計画
現在 url フィールドに公式サイトを入れているが、
Amazonの商品ページに変更するだけでアフィリエイト化できる。

```json
// 現在
"url": "https://kurosaki-knife.com/"

// 変更後（Amazonアフィリエイトリンク）
"url": "https://www.amazon.com/dp/[ASIN]?tag=[アフィリエイトID]"
```

## 次のSTEP（STEP 5：個別ページ生成）

data.jsonの各エントリから個別HTMLページを生成する。

```
items/yu-kurosaki.html
items/sakai-takayuki.html
...（件数分）
```

**SEO効果：**
- 「Yu Kurosaki knife」「Sakai Takayuki review」等で個別ページが検索結果に出る
- 現在の1枚HTMLでは狙えない細かいキーワードをカバーできる

**生成コスト目安：**
- 30件のHTMLをClaude APIで生成 → 約30〜50円
- 100件に拡張しても約100〜150円

## 今すぐ動作確認する方法

1. `knife-db` フォルダを開く
2. `index.html` をダブルクリック（ブラウザで開く）
3. フィルターをクリックして動作確認
4. ※ ローカルファイルからdata.jsonを読む場合、ブラウザのCORS制限に引っかかる場合あり
   → その場合はVSCodeのLive Server拡張機能を使うか、Vercelに上げて確認する

## Vercelへの公開手順（STEP 6の予習）

```
1. GitHub アカウントを作成（無料）
2. knife-db フォルダを GitHub リポジトリとして作成
3. vercel.com でサインイン → Import Repository
4. デプロイボタンを押す
5. 数分で https://[your-name].vercel.app に公開完了
```
